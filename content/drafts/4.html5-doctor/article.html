With the gigantic leaps forward that Mobile Safari has given us, all of a sudden the potential for building entire applications out of the holy trinity of html/css/javascript is become a serious possibility.

We can see frameworks tailored towards this goal beginning to spring up and even OSX applications themselves are starting to integrate these technologies.

So, being the html-minded developers that we are, we really wanted to get into some of the newer parts of the html5 spec and see if we could use them to create a mobile app worthy of competing with the native iOS apps.

... What does the app ACTUALLY do?

During the process of building the app we discovered some interesting techniques that we hadn't seen before. And because the app itself is so simple, we decided to open-source it and highlight a few interesting points.

TAKE-AWAY! The first real benefit of designing a standalone html app is that you are writing for a single, modern browser. For anyone who has been supporting IE6 for the past 4 years, this is a huge one. CSS3 selectors, seriously powerful native javascript & all the html5 trimmings.

TAKE-AWAY! It's easy to forget that one of the major reasons behind the popularity of the vast array of javascript libraries is to add support for modern javascript features to older browsers. Being able to use querySelectorAll, XMLHttpRequest and the native JSON parser removes need for a js framework at all for many projects.
